Introduction

HTML (HyperText Markup Language) is the standard markup language for creating web pages. It defines the structure of web content using a system of tags and attributes.

Purpose of HTML

a. Structuring Content: HTML allows developers to organize and structure content into meaningful components like headings, paragraphs, lists, and tables.
b. Cross-Browser Support: HTML is universally supported across web browsers.
c. SEO & Accessibility: HTML tags help search engines and assistive technologies understand the content.

<!DOCTYPE html>

The <!DOCTYPE html> declaration is an instruction to the web browser about the version of HTML that the page is written in. It ensures that the browser renders the page correctly according to the specified standards.

It is not a tag but a declaration, and it must appear at the very top of an HTML document, before the <html> tag.

Purpose of <!DOCTYPE html>
  • Ensures the browser renders the HTML document in standards mode instead of quirks mode.
  • Specifies the version of HTML being used, such as HTML5.
  • Improves cross-browser compatibility and ensures consistent rendering of the page.
HTML5 <!DOCTYPE> Declaration

The modern <!DOCTYPE> declaration for HTML5 is:

<!DOCTYPE html>

This is a simplified declaration compared to older versions of HTML and applies to all HTML5 documents.

Older <!DOCTYPE> Declarations

Before HTML5, <!DOCTYPE> declarations were more complex and looked like this:

  • HTML 4.01 Strict:
    <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
  • HTML 4.01 Transitional:
    <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">
  • XHTML 1.0 Strict:
    <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">

These older declarations were used to specify different Document Type Definitions (DTDs) for validation.

What Happens Without <!DOCTYPE>?

If you omit the <!DOCTYPE> declaration:

  • The browser switches to quirks mode, where it uses outdated rendering methods to display the page.
  • This can lead to inconsistent behavior across different browsers.
  • Modern HTML5 features may not work as expected.
Best Practices
  • Always include the <!DOCTYPE html> declaration at the top of your HTML document.
  • Use the simplified HTML5 declaration for modern web development.
  • Ensure your HTML is valid and follows the latest web standards.

<html> Tag

What is the <html> Tag?

The <html> tag is the root element of an HTML document. It represents the entire HTML document and serves as the container for all other elements, including metadata and content.

All HTML documents must start with the <html> tag and include a closing </html> tag at the end.

Structure of an HTML Document

A basic HTML document structure:

<!DOCTYPE html>
<html lang="en">
    <head>
        <title>Sample Document</title>
    </head>
    <body>
        <h1>Welcome to HTML</h1>
    </body>
</html>
Attributes of <html> Tag

The <html> tag supports the following attributes:

  • lang: Specifies the language of the document. It is essential for accessibility and SEO.
    <html lang="en">
  • dir: Specifies the direction of text within the document.
    • ltr: Left-to-right (default).
    • rtl: Right-to-left (e.g., Arabic, Hebrew).
    • auto: Automatically determines the direction based on the content.

    <html dir="rtl">
  • xmlns: Specifies the XML namespace for the document. It is used when working with XHTML.
    <html xmlns="http://www.w3.org/1999/xhtml">
Using the <html> Tag with Attributes

Examples of the <html> tag with various attributes:

  • Default Usage: 
    <html lang="en">
    ...
</html>
  • Right-to-Left Language (Arabic): 
    <html lang="ar" dir="rtl">
    ...
</html>
  • With XML Namespace: 
    <html xmlns="http://www.w3.org/1999/xhtml">
    ...
</html>
Common Use Cases
  • Specifying the language of the document for accessibility and SEO.
  • Defining the text direction for languages like Arabic or Hebrew.
  • Providing XML namespaces for compatibility with XHTML or other XML-based formats.
Best Practices
  • Always specify the lang attribute for better accessibility and search engine optimization.
  • Set the dir attribute appropriately for right-to-left or left-to-right languages.
  • Ensure that the xmlns attribute is used only when necessary for XML-based standards.

<head> Tag

What is the <head> Tag?

The <head> tag is a container for metadata (data about data) and links to external resources in an HTML document.

It is placed between the <html> tag and the <body> tag and is not visible on the webpage but plays a critical role in setting up the document.

Structure of the <head> Tag

An example of how the <head> tag fits into the HTML document structure:

<!DOCTYPE html>
<html lang="en">
    <head>
        <title>My Document</title>
        <meta charset="UTF-8">
        <link rel="stylesheet" href="styles.css">
        <script src="script.js"></script>
    </head>
    <body>
        ...
    </body>
</html>
Tags Inside the <head> Tag

<title> Tag:

What is the <title> Tag?

The <title> tag defines the title of the document, which is displayed in the browser's title bar or tab. It is placed inside the <head> section of an HTML document.

The title is an essential part of a webpage as it provides the page's identity and is used by search engines to display page results. It also appears when the user bookmarks a page.

Structure of the <title> Tag

The <title> tag is placed inside the <head> section of the HTML document:

<html>
    <head>
        <title>My Webpage</title>
    </head>
    <body>
        ...
    </body>
</html>
Attributes of the <title> Tag

The <title> tag does not have any specific attributes. Its content defines the title text that will be displayed in the browser's title bar or tab.

Common Uses of the <title> Tag
  • Webpage Identity: The title of a webpage is displayed in the browser tab. It helps users identify the page they are on and is visible when they bookmark the page.
    <title>Welcome to My Website</title>
  • SEO (Search Engine Optimization): Search engines use the title tag to rank the page and understand its content, so including relevant keywords is important for SEO.
    <title>Best Pizza in New York - Delicious Pizza & Pasta</title>
  • Bookmarks: When users add a webpage to their bookmarks or favorites, the title is used to display in the list.
    <title>JavaScript Tutorial - Learn JavaScript Easily</title>
Examples of Usage
  • Simple Title: 
    <title>My Website</title>
  • SEO-Optimized Title with Keywords: 
    <title>Buy Affordable Laptops Online - Best Deals</title>
  • Title with Dynamic Content:
    In JavaScript, you can dynamically change the title of the page: 
    document.title = 'New Page Title';
  • Title for Social Sharing: When sharing a link on social media, a well-crafted title can encourage clicks. Titles should be compelling and relevant: 
    <title>Get the Best Travel Deals â€“ Exclusive Offers!</title>
Best Practices for Using the <title> Tag
  • Keep titles concise but descriptive (50-60 characters is ideal for SEO).
  • Use relevant keywords that describe the content of the page but avoid keyword stuffing.
  • Include the brand name or website name in the title, especially for homepage and landing pages.
  • Ensure that each page on your website has a unique title to differentiate content.
  • Consider user intent and make the title engaging and informative.

<meta> Tag:

What is the <meta> Tag?

The <meta> tag is used to provide metadata about the HTML document. Metadata is information about the document that is not directly visible to the user but is used by browsers, search engines, and other web services. The <meta> tag is placed within the <head> section of the document.

Structure of the <meta> Tag

The <meta> tag does not have a closing tag and is typically self-closing:

<html>
    <head>
        <meta name="description" content="Learn HTML and CSS">
    </head>
    <body>
        ...
    </body>
</html>
Attributes of the <meta> Tag

The <meta> tag has several attributes, each serving different purposes. Below are some common attributes:

  • name: Specifies the name of the metadata.
  • content: Defines the value of the metadata.
  • http-equiv: Provides HTTP headers information (used for setting document refresh or character encoding).
  • charset: Specifies the character encoding for the document.
Common Uses of the <meta> Tag
  • Document Metadata: Providing general information about the document, such as description, author, and keywords: 
    <meta name="description" content="Learn HTML, CSS, and JavaScript">
  • SEO (Search Engine Optimization): Search engines use the <meta> tag to understand the content of the page and index it accordingly: 
    <meta name="keywords" content="HTML, CSS, JavaScript, Web Development">
  • Viewport Settings: Adjusts the viewport settings on mobile devices for better responsiveness: 
    <meta name="viewport" content="width=device-width, initial-scale=1">
  • Character Encoding: Specifies the character encoding for the document: 
    <meta charset="UTF-8">
  • Author Information: Specifies the author of the document: 
    <meta name="author" content="John Doe">
  • Document Refresh: Automatically refreshes the document after a specific time interval: 
    <meta http-equiv="refresh" content="30">
Examples of Using <meta> Tag
  • Character Set Declaration: 
    <meta charset="UTF-8">
  • Document Description for SEO: 
    <meta name="description" content="Learn how to build a website using HTML and CSS">
  • Viewport Settings for Mobile Responsiveness: 
    <meta name="viewport" content="width=device-width, initial-scale=1">
  • Automatic Page Refresh: 
    <meta http-equiv="refresh" content="5">
  • Content-Type Header: 
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
Best Practices for Using <meta> Tag
  • Use <meta charset="UTF-8"> to ensure your page supports all characters, including special characters.
  • Always include a <meta name="viewport" tag for better mobile responsiveness.
  • Keep the <meta name="description" tag concise (about 150 characters) to improve SEO.
  • Use <meta name="keywords" for better search engine indexing.
  • Avoid overusing <meta http-equiv="refresh" as it may confuse search engines or users.

<link> Tag:

What is the <link> Tag?

The <link> tag is used to link external resources like stylesheets, icons, and more to an HTML document. It is typically placed within the <head> section and helps associate external files with the webpage.

Structure of the <link> Tag

The <link> tag does not have a closing tag and is usually self-closing:

<html>
    <head>
        <link rel="stylesheet" href="styles.css">
    </head>
    <body>
        ...
    </body>
</html>
Attributes of the <link> Tag

The <link> tag has several attributes that help specify the relationship between the document and the linked resource. Some of the most common attributes are:

  • rel: Specifies the relationship between the current document and the linked resource. It is commonly used to indicate a "stylesheet" or "icon".
  • href: Specifies the URL of the external resource.
  • type: Specifies the media type of the linked resource. This is often used for stylesheets to define the MIME type.
  • media: Specifies the media type for which the linked resource is optimized (e.g., print, screen).
  • sizes: Defines the size of the linked resource, commonly used for icon images.
  • hreflang: Specifies the language of the linked resource.
Common Uses of the <link> Tag
  • Linking External Stylesheets: The <link> tag is most commonly used to link external CSS files to an HTML document: 
    <link rel="stylesheet" href="styles.css">
  • Adding Icons (Favicon): The <link> tag is used to specify the favicon (the icon shown in the browser tab): 
    <link rel="icon" href="favicon.ico">
  • Linking to Web Fonts: The <link> tag is used to link to external web fonts, such as Google Fonts: 
    <link rel="stylesheet" href="https://fonts.googleapis.com/css?family=Roboto">
  • Adding Alternate Stylesheets: You can define alternative stylesheets with different media types (e.g., for printing): 
    <link rel="stylesheet" media="print" href="print.css">
  • Specifying Icons for Various Devices: For responsive web design, the <link> tag can be used to link to different icons for various device sizes: 
    <link rel="icon" sizes="16x16" href="icon-16x16.png">
Examples of Using <link> Tag
  • Linking an External CSS File: 
    <link rel="stylesheet" href="styles.css">
  • Adding a Favicon Icon: 
    <link rel="icon" href="favicon.ico">
  • Linking to Google Fonts: 
    <link rel="stylesheet" href="https://fonts.googleapis.com/css?family=Roboto">
  • Specifying a Print Stylesheet: 
    <link rel="stylesheet" media="print" href="print.css">
  • Adding a Larger Icon for Devices: 
    <link rel="icon" sizes="32x32" href="icon-32x32.png">
Best Practices for Using <link> Tag
  • Always use rel="stylesheet" when linking CSS files for improved clarity.
  • Use rel="icon" to specify the website's favicon.
  • For responsive designs, use media="screen" to define stylesheets specifically for different devices (mobile, tablet, etc.).
  • Place the <link> tag inside the <head> section for proper document structure.
  • If using an external resource (like a font), make sure to include rel="stylesheet" and ensure the link is accessible.

<script> Tag:

What is the <script> Tag?

The <script> tag is used to embed or reference executable JavaScript code within an HTML document. It can be placed in both the <head> and <body> sections. This tag is essential for adding interactivity and dynamic behavior to web pages.

Structure of the <script> Tag

The <script> tag can be written in two main ways:

  • Inline JavaScript: The script is written directly between the opening and closing <script> tags.
    • <script>
    console.log('Hello, World!');
</script>
  • External JavaScript: The script is linked from an external file using the src attribute.
    • <script src="path/to/your-script.js"></script>
Attributes of the <script> Tag

The <script> tag has several attributes that control how the script is loaded and executed. The most common attributes are:

  • src: Specifies the URL of an external script file.
  • type: Specifies the scripting language used. The default value is text/javascript, but for modern browsers, it is optional as JavaScript is the default scripting language.
  • async: If present, the script will be executed asynchronously, meaning it will not block the rendering of the page.
  • defer: If present, the script will be executed after the document has finished parsing, but before the DOMContentLoaded event.
  • integrity: Used to provide a hash value for the script, ensuring that it has not been tampered with.
  • crossorigin: Specifies how to handle cross-origin requests for the script.
  • nomodule: Specifies that the script should only be executed if the browser does not support modules.
Common Uses of the <script> Tag
  • Embedding JavaScript Code: The <script> tag is used to directly embed JavaScript code in an HTML document.
  • Linking External JavaScript Files: The <script src="path/to/file.js"> tag is used to link to external JavaScript files.
  • Asynchronous Loading: Use async to load scripts asynchronously so that they do not block page rendering.
  • Deferred Execution: Use defer to ensure that the script is executed after the HTML document has been parsed.
  • Module-based Scripts: The type="module" attribute is used for JavaScript modules, which allow for the import and export of code between different JavaScript files.
Examples of Using <script> Tag
  • Inline JavaScript Example: 
    <script>
    console.log('Inline JavaScript is running!');
</script>
  • Linking an External JavaScript File: 
    <script src="app.js"></script>
  • Asynchronous Script Example: 
    <script src="app.js" async></script>
  • Deferred Script Example: 
    <script src="app.js" defer></script>
  • Using Modules: 
    <script type="module" src="app.js"></script>
Best Practices for Using <script> Tag
  • Place <script> tags just before the closing </body> tag to allow the page to load first, enhancing page load performance.
  • If using external scripts, always specify the src attribute.
  • Use async or defer for external scripts to improve page loading and prevent blocking.
  • For modern web applications, consider using JavaScript modules (type="module") to improve code organization and reuse.
  • Use integrity and crossorigin to enhance security when loading third-party scripts.

<style> Tag:

What is the <style> Tag?

The <style> tag is used to define CSS rules within an HTML document. It is typically placed within the <head> section, where it can apply styles globally to the HTML elements in the document. Using the <style> tag, developers can write CSS code directly inside the HTML page.

Structure of the <style> Tag

The <style> tag is used to write CSS code directly within an HTML document. Here is how it is typically written:

<head>
    <style>
        body {
            background-color: lightblue;
        }
        h1 {
            color: green;
        }
    </style>
</head>

CSS rules within the <style> tag are written just like regular CSS code, with selectors and properties.

Attributes of the <style> Tag

The <style> tag does not have many attributes, but the most commonly used are:

  • type: Specifies the MIME type of the content, typically set to text/css to indicate the CSS language. This is optional, as browsers default to this value.
  • media: Defines the media for which the style sheet is intended (e.g., screen, print, etc.). This is useful for responsive design and when different styles are needed for various devices.
  • scoped: An optional attribute that, when added, limits the scope of the style rules to the element that contains the <style> tag. However, this attribute is no longer widely supported and is considered deprecated in modern HTML standards.
Common Uses of the <style> Tag
  • Internal CSS: The <style> tag is used to define styles inside an HTML document without linking to an external CSS file.
  • Responsive Design: The <style> tag can include media queries that allow for adaptive layouts depending on the screen size.
  • Quick Styling: When you need to apply quick, temporary styles to an HTML document, you can use the <style> tag directly in the document.
Examples of Using the <style> Tag
  • Basic Internal CSS Example: 
    <head>
    <style>
        body {
            font-family: Arial, sans-serif;
            background-color: #f0f0f0;
        }
    </style>
</head>
  • Responsive CSS with Media Queries: 
    <head>
    <style>
        body {
            background-color: lightblue;
        }
        @media (max-width: 600px) {
            body {
                background-color: pink;
            }
        }
    </style>
</head>
  • Scoped Style (Deprecated): 
    <div scoped>
    <style>
        p {
            color: red;
        }
    </style>
    <p>This text will be red</p>
</div>

    Note: The scoped attribute is deprecated and should not be used in modern HTML.

Best Practices for Using the <style> Tag
  • Keep CSS separate from HTML by using external stylesheets for larger projects to improve maintainability and code separation.
  • Use the <style> tag for quick, small changes or when testing out CSS directly in an HTML document.
  • Make use of media queries inside the <style> tag to ensure the page looks good on all devices.
  • Be cautious when using inline styles directly in HTML elements, as this can make your code harder to manage.

<base> Tag:

What is the <base> Tag?

The <base> tag specifies a base URL for relative links within a document. It is placed inside the <head> section and provides a way to set a base URL for all relative URLs in the document, making it easier to manage and link resources.

Structure of the <base> Tag

The <base> tag does not have any closing tag and typically looks like this:

<head>
    <base href="https://www.example.com/">
</head>

All relative URLs in the document will now resolve relative to https://www.example.com/.

Attributes of the <base> Tag

The <base> tag has the following main attributes:

  • href: Specifies the base URL for all relative URLs in the document. It is a required attribute when using the <base> tag.
  • target: Specifies the default target for all links in the document. This determines where linked documents will open. Possible values are:
    • _self: Opens the link in the same frame (default).
    • _blank: Opens the link in a new window or tab.
    • _parent: Opens the link in the parent frame.
    • _top: Opens the link in the full body of the window.
Common Uses of the <base> Tag
  • Setting a Global Base URL: You can set the <base> tag to define a global base URL for all relative links on the page, making it easier to manage and maintain links.
  • Managing Links in Single-Page Applications (SPA): In modern web applications, <base> can be used to manage relative links in SPA frameworks, ensuring consistency across the application.
  • Improving Portability of Relative Links: If you move a document to a different directory or location, using the <base> tag ensures that all links still work correctly without needing to update individual URLs.
Examples of Using the <base> Tag
  • Setting a Base URL: 
    <head>
    <base href="https://www.example.com/">
</head>

<a href="page.html">Go to Page</a>

    In this case, the link page.html will be resolved relative to https://www.example.com/.

  • Setting a Base URL and Target: 
    <head>
    <base href="https://www.example.com/" target="_blank">
</head>

<a href="page.html">Go to Page</a>

    Now, the link will open in a new window or tab because of the target="_blank" attribute.

Best Practices for Using the <base> Tag
  • Use the <base> tag when your site relies heavily on relative URLs, especially when migrating a document to different directories.
  • Be cautious when using the <base> tag in large websites. It can affect the behavior of all relative links, which might result in unexpected behavior if used incorrectly.
  • For single-page applications (SPAs), the <base> tag can help with routing and maintaining a consistent URL structure for the app.
Examples of Usage
  • Default Document Setup: 
    <head>
    <title>My Webpage</title>
    <meta charset="UTF-8">
    <link rel="stylesheet" href="styles.css">
</head>
  • Viewport for Responsive Design:
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
  • Adding Favicon:
    <link rel="icon" href="favicon.ico" type="image/x-icon">
  • SEO Metadata:
    <meta name="description" content="This is a sample website.">
Best Practices
  • Always include a <title> tag with a descriptive title.
  • Use <meta charset="UTF-8"> for proper character encoding.
  • Leverage the <meta> viewport tag for responsive design on mobile devices.
  • Link external stylesheets and scripts using <link> and <script> for better modularity.
