Function Declaration, Expression, and Arrow Functions

Recursion with Function Declaration

Write a recursive function declared using a **function declaration** to calculate the Fibonacci series up to a given number. The input should be taken from an HTML form, and the results should be displayed back in HTML

Function Expression for Sorting

Create a **function expression** to sort an array of numbers. The array should be taken as inputs from an HTML form, and the sorted array should be displayed in a table on the HTML page.

Arrow Function for Event Handling

Implement a button in HTML that uses an **arrow function** to handle its click event. The function should update a counter displayed on the HTML page every time the button is clicked.

Immediately Invoked Function Expressions (IIFE)

Create an **IIFE** (Immediately Invoked Function Expression) to initialize a set of default values on the page. For example, it should set default text in an input box.

Parameters, Arguments, and Default Values

Function with Default Paramenters

Write a function that calculates the area of a rectangle. Use default values for length and width if they are not provided. The user should input values through an HTML form, and the result should display back on the page.

Rest Parameters for variable Arguments

Create a function that takes a dynamic number of arguments using the rest parameter. The function should return the sum of all arguments passed. Take input from an HTML form as a comma-separated list and display the sum on the page.

Nested Functions with Default Values

Create a nested function setup where the outer function calculates the perimeter of a rectangle using default values for length and width. The inner function calculates the area of the rectangle using the same parameters. Display both results on the page based on user input or defaults..

Callback Functions with Default Parameters

Write a function that accepts another function as a parameter (a callback). The callback function should take a default value if not provided and process user input (e.g., convert a string to uppercase). Show the final processed string on the page.

Closures and Lexical Scoping

Private variables with Closures

Write a function that acts as a bank account. The function should have private variables for balance and methods to deposit, withdraw, and check balance. Ensure balance manipulation is only possible through these methods. Add HTML inputs for user actions.

Lexical Scoping in Nested Functions

Write a function that has three levels of nested functions. The innermost function should access variables defined in the outer functions and return a concatenated string. Allow the user to set the variables in the HTML form and display the output.

Cachning Results

Create a function that caches the results of expensive calculations. Use a closure to store previously computed results. For example, a function that calculates the factorial of a number should cache the results for faster computation of repeated inputs. Allow the user to input numbers and display results on the page.

Intermediate Topics

High-Order Functions

High Order Function with Arrow Functions

Use an **arrow function** as the callback to filter an array of numbers (e.g., removing all numbers greater than a user-defined limit). The input should come from an HTML form, and the filtered array should be displayed back to the HTML page.
