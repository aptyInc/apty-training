What is JavaScript?

JavaScript is a high-level, interpreted, and dynamic programming language primarily used for creating interactive effects within web browsers. It enables developers to manipulate web pages by interacting with the Document Object Model (DOM), responding to user inputs, handling events, and performing other tasks that make websites dynamic and responsive.

JavaScript is one of the core technologies of the web, along with HTML (structure) and CSS (style), and is supported by all modern web browsers.

Why is JavaScript Important
  • Interactivity: JavaScript makes web pages interactive by enabling features such as form validation, dynamic content loading, animations, interactive maps, and more.
  • Client-Side Scripting: JavaScript executes in the browser, enabling operations on the client side (user's machine), making websites feel more responsive and interactive without constant communication with the server.
  • Browser Control: JavaScript interacts with the DOM, which is the internal representation of the HTML structure, allowing developers to manipulate the content, layout, and behavior of web pages dynamically.
  • Asynchronous Operations: JavaScript allows for asynchronous operations (e.g., AJAX) that enable data fetching and content updates without requiring a page reload, improving the user experience.
  • Widely Supported: JavaScript is universally supported by all browsers and is essential for modern web development, particularly for creating single-page applications (SPAs) and progressive web apps (PWAs).
How Does JavaScript Get Executed in the Browser?

1. Loading HTML Page:
The browser begins by loading an HTML page. As it parses the HTML, it constructs the DOM, representing the structure of the page.

2. Finding and Loading JavaScript:
When the browser encounters a <script> tag in the HTML, it processes the JavaScript code. The script may be inline (within the HTML) or external (linked via the src attribute).

3. Execution by JavaScript Engine:
The browser has a built-in JavaScript engine (such as V8 in Chrome, SpiderMonkey in Firefox, or JavaScriptCore in Safari) that interprets and executes JavaScript code.

4. Execution Context and Call Stack:
The browser creates an execution context for JavaScript execution, which includes the global execution context (for global code) and function execution contexts (for code inside functions).

5. Interaction with the DOM and Rendering:
As JavaScript executes, it interacts with the DOM, modifying elements, adding or removing nodes, or changing attributes.

6. Event Handling:
JavaScript can also listen for events (e.g., clicks, keypresses, form submissions) and trigger functions in response, allowing for interactivity and user input handling.

Different Ways of Writing JavaScript

1. Inline JavaScript:

<html>
    <head>
        <body>
            <button onclick="alert('Hello, welcome to Inline JavaScript!')">Click Me</button>
        </body>
    <head>
</html>

2. Internal JavaScript:

<html>
    <head>
        <script>
            // JavaScript code here
            console.log('Hello from internal JS!');
        </script>
    </head>
</html>

3. External JavaScript:

<script src="path/to/your-script.js"></script>
